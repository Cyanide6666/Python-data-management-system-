# Python data management system: Sistema simple de Gestión de Datos de Empleados

## Descripción del proyecto
Este sistema permite la manipulación de datos de manera eficiente utilizando las mejores prácticas de desarrollo.

## Tecnologías Utilizadas
-Lenguaje: Python 3

-Estructuras de Datos: Uso de listas, diccionarios, tuplas y conjuntos para el manejo eficiente de información

-Estructuras de Control: Implementación de bucles while, condicionales if/else y manejo de excepciones

-Algoritmos: Uso de funciones recursivas para cálculos específicos de nómina

-Modularización: Organización en múltiples archivos .py para mejorar la reusabilidad del código

## Estructuras de datos utilizadas
-Listas (list): Funcionan como el contenedor dinámico principal para almacenar la colección completa de registros.

-Diccionarios (dict): Se utilizan para representar a cada empleado de forma estructurada, almacenando ID, Nombre, Puesto y Salario.

-Tuplas (tuple): Para definir elementos inmutables, como los encabezados de las tablas de salida.

-Conjuntos (set): Aplicados para asegurar la unicidad de categorías o departamentos y evitar elementos duplicados en el procesamiento


## Características y Funcionalidades
El sistema cumple con los siguientes pilares de desarrollo:
- Modularización: Separación de la lógica de negocio (gestion.py) de la interfaz de usuario (main.py).

- Estructuras de Datos: Implementación de **Listas** para colecciones y **Diccionarios** para la representación de objetos de datos.
  
- Recursividad: Uso de funciones recursivas para realizar cálculos como la sumatoria de salarios.
  
- Entrada/Salida de Datos: Persistencia mediante lectura de archivos de texto (.txt) y salida formateada profesionalmente con **f-strings**.
  
- Control de Flujo: Estructuras condicionales y bucles robustos para la navegación en el sistema.

## Estructura del Repositorio
- main.py: Punto de entrada de la aplicación y manejo de menús.
  
- gestion.py: Módulo que contiene la lógica de procesamiento y funciones recursivas.
  
- datos.txt: Archivo fuente de datos para pruebas.
  
- docs/: Carpeta con la documentación técnica y pruebas de la visualización del programa.
